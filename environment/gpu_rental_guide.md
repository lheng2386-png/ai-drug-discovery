# GPU 租用操作单

更新日期：2026-07-09

## 推荐平台与显卡

第一选择：**AutoDL + 单张 RTX 3090 24 GB**。

理由：

1. RTX 3090 的 Ampere 架构与 TargetDiff 官方 CUDA 11.6 环境匹配；
2. 24 GB 显存足以作为 checkpoint 推理和单口袋 smoke test 的合理起点；
3. AutoDL 当前公开参考价约 ¥1.32/GPU·小时，低于其 4090 和 5090；
4. 国内连接、镜像和按小时计费对入门较方便。

公开价格会变化，下单页价格为准：<https://www.autodl.com/home>

## 下单配置

| 项目 | 选择 |
|---|---|
| GPU | RTX 3090 24 GB，单卡 |
| CPU | 8 核以上 |
| 内存 | 32 GB 最低；优先 64 GB |
| 镜像 | Ubuntu 20.04 |
| CUDA | 11.6；若没有，选 PyTorch 1.13 + 最接近的 CUDA 11.x 镜像后核验 |
| 系统盘 | 平台默认即可 |
| 数据盘 | 100 GB 起步 |
| 计费 | 按量/按小时 |
| 自动关机 | 开启，建议先设 2–3 小时 |

不要选：

- Windows 镜像；
- 多卡实例；
- 12 GB 或更小显存；
- 第一轮就购买包月；
- 第一轮就扩到很大的数据盘。

## 预算

建议首次充值/预算控制在 **20–30 元**：

- 1 小时：检查机器和驱动；
- 1–2 小时：创建 Conda 环境和安装依赖；
- 1–2 小时：下载最小资源并运行 smoke test；
- 预留失败重试时间。

安装依赖和下载数据时如果平台支持“无卡模式”，应关闭 GPU 计费后再操作。

## 创建实例后

先不要安装任何包。在终端执行：

```bash
uname -a
cat /etc/os-release
nvidia-smi
df -h
free -h
python3 --version
conda --version
git --version
```

将完整输出保存下来。需要确认：

- GPU 确实为 RTX 3090；
- 显存约 24 GB；
- NVIDIA 驱动正常；
- 可用磁盘不少于 80 GB；
- 内存不少于 32 GB。

## 何时升级显卡

- 单次采样发生可复现的 CUDA OOM，且降低 batch/口袋规模后仍失败：换 A6000/A40 48 GB；
- 进入 ESM-2 大模型或大 batch 训练：按实测显存换 48/80 GB；
- 使用要求 CUDA 12.8+ 的现代模型：可换 RTX 5090；
- 不因“更快”提前升级，先确认瓶颈确实在 GPU。

## 费用安全

- 每次完成操作后关机并确认停止计费；
- 数据盘、镜像和公网流量可能单独收费；
- 不在云端保存 API key、GitHub token 或真实药物/患者敏感数据；
- 重要配置和日志同步回本地，模型大文件只记录来源与校验值。
